# Introduction

Welcome to the **CS698 Project Documentation Hub**. This site serves as the "source of truth" for the application's architecture, development practices, and API specifications.

## Quick Start

If you are a new developer joining the project, start here:

1.  **[Architecture](./architecture)**: Understand the layered structure and core layers.
2.  **[State Management](./state-management)**: Learn how we handle global and ephemeral state.
3.  **[Testing](./testing)**: Review the Robot Pattern and stability rules for Windows.
4.  **[Security](./security)**: Understand data redaction and network security protocols.
5.  **[API Reference](/api)**: Browse the interactive API documentation powered by Scalar.

## Core Principles

- **Immutability First**: Domain models are immutable.
- **Tests as Documentation**: Integration tests (Robots) describe intended behavior.
- **Premium Aesthetics**: UI must feel polished, responsive, and follow the [Design System](./design-system).
