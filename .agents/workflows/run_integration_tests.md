---\r\ndescription: How to run Flutter integration tests on Windows or Linux (devcontainer) with proper logging and headless GUI support.\r\n---\r\n\r\n# Running Integration Tests (Standardized Methodology)\r\n\r\nThis workflow covers running integration tests on **Windows** and **Linux/Devcontainer**. The Dart runner auto-detects the platform and uses the correct device (`-d windows` or `-d linux`) and display server (`xvfb-run` on Linux).\r\n\r\n## 1. Execution (Dart CLI)\r\n\r\nTests run sequentially to prevent resource locks on the debug port or `.dart_tool` cache.\r\n\r\n// turbo\r\n1. Run the dedicated Dart tool:\r\n```bash\r\n# Basic execution (works on Windows AND Linux/Devcontainer):\r\ndart run tool/run_integration_tests.dart\r\n\r\n# Execution with file logging (Recommended):\r\ndart run tool/run_integration_tests.dart --log-file integration_report.log\r\n```\r\n\r\n**Platform behavior:**\r\n- **Windows**: Uses `-d windows` with native display.\r\n- **Linux / Devcontainer**: Uses `-d linux` under `xvfb-run --auto-servernum` for headless GUI testing via a virtual framebuffer.\r\n\r\n> [!IMPORTANT]\r\n> **Devcontainer Prerequisite**: The devcontainer `Dockerfile` must have `xvfb` installed. This is already configured in the project's `.devcontainer/Dockerfile`.\r\n\r\n> [!IMPORTANT]\r\n> **Non-Destructive Methodology**: Do NOT use `taskkill` or forcefully terminate `dart.exe` or `flutter.exe` processes manually. The Dart tool manages child processes synchronously.\r\n\r\n## 2. Diagnostics & Live Tracing (MCP + Native Tools)\r\n\r\nWhen an integration test fails (`No element found`) and you need a \"live trace\":\r\n\r\n1. **Native Flutter Diagnostics**:\r\n   ```dart\r\n   debugDumpApp(); // Dumps entire widget tree to console\r\n   debugDumpRenderTree(); // Dumps render layout constraints\r\n   ```\r\n   Or capture a UI snapshot:\r\n   ```dart\r\n   final binding = IntegrationTestWidgetsFlutterBinding.ensureInitialized();\r\n   await binding.takeScreenshot('failure_trace_1');\r\n   ```\r\n\r\n2. **Using dart-mcp-server**:\r\n   Use `mcp_dart-mcp-server_run_tests` internally, or launch the app in debug mode with `mcp_dart-mcp-server_launch_app` and connect for live traces.\r\n\r\n3. **Using marionette MCP**:\r\n   Start the app with `flutter run -d windows` (or `-d linux` under `xvfb-run`), locate the VM service URI, then use `mcp_marionette_connect`, `mcp_marionette_take_screenshots`, etc.\r\n
